# SKUD Admin Panel

Админ-панель находится в `admin/` и раздается FastAPI по адресу:

```text
http://localhost:8000/admin/
```

Запуск:

```bash
pip install -r requirements.txt
uvicorn api.app:app --reload --host 0.0.0.0 --port 8000
```

Корневой адрес `http://localhost:8000/` автоматически перенаправляет на `/admin/`.

## Общий принцип

Админка - это статический клиент к REST API:

- `admin/index.html` - структура интерфейса;
- `admin/styles.css` - белый рабочий UI;
- `admin/app.js` - логика запросов, состояние, формы и таблицы.

Ключ API хранится в `localStorage` браузера:

- `skud.apiBase` - базовый URL API;
- `skud.apiKey` - админский `X-API-Key`.

## Первичная настройка

Раздел `Настройка` позволяет создать компанию с нуля:

1. Ввести URL API, например `http://localhost:8000`.
2. Ввести название компании.
3. Ввести slug компании.
4. Ввести `SKUD_BOOTSTRAP_TOKEN`.
5. Нажать `Создать и подключить`.

После успешного создания сервер возвращает `admin_api_key`, а админка сразу сохраняет его и подключается к новой компании.

## Разделы

### Обзор

Показывает:

- количество сотрудников;
- количество помещений;
- сколько человек сейчас внутри;
- сколько событий прохода было сегодня;
- текущую занятость помещений;
- последние события прохода;
- проходимость по дням и помещениям.

Используемые API:

- `GET /api/v1/employees`
- `GET /api/v1/rooms`
- `GET /api/v1/access-events`
- `GET /api/v1/stats/occupancy`
- `GET /api/v1/stats/throughput`
- `GET /api/v1/stats/office-time`

### Сотрудники

Функции:

- добавить сотрудника;
- смотреть список сотрудников;
- искать сотрудника по ФИО, табельному ID, должности или email;
- активировать или временно отключить сотрудника;
- загрузить фото лица для face-профиля;
- выпустить QR-пропуск и получить payload для QR.

Используемые API:

- `POST /api/v1/employees`
- `GET /api/v1/employees`
- `PATCH /api/v1/employees/{employee_id}`
- `POST /api/v1/employees/{employee_id}/face-photos`
- `POST /api/v1/employees/{employee_id}/qr-passes`

### Помещения

Функции:

- добавить помещение;
- указать код помещения;
- указать вместимость;
- указать описание;
- включить или выключить помещение;
- смотреть текущую занятость.

Используемые API:

- `POST /api/v1/rooms`
- `GET /api/v1/rooms`
- `PATCH /api/v1/rooms/{room_id}`
- `GET /api/v1/stats/occupancy`

### Доступы

Функции:

- связать сотрудника с помещением;
- выбрать методы прохода: `qr`, `face` или оба;
- задать дату начала действия правила;
- задать дату окончания действия правила;
- включить или отключить правило;
- фильтровать правила по сотруднику и помещению.

Используемые API:

- `POST /api/v1/access-rules`
- `GET /api/v1/access-rules`
- `PATCH /api/v1/access-rules/{rule_id}`

### Сканеры

Функции:

- создать сканер;
- привязать сканер к помещению;
- выбрать направление: `entry` или `exit`;
- выбрать поддерживаемые методы: `qr`, `face` или оба;
- включить или выключить сканер;
- увидеть последний сигнал сканера;
- получить `scanner_token` сразу после создания.

Важно: токен сканера показывается только один раз после создания. Сервер хранит только хеш токена.

Используемые API:

- `POST /api/v1/scanners`
- `GET /api/v1/scanners`
- `PATCH /api/v1/scanners/{scanner_id}`

### Журнал

Функции:

- смотреть события прохода;
- фильтровать события по сотруднику;
- фильтровать события по помещению;
- фильтровать события по решению: `granted` или `denied`;
- видеть метод, причину решения, сканер и время события.

Используемые API:

- `GET /api/v1/access-events`
- `GET /api/v1/employees`
- `GET /api/v1/rooms`

## Ограничения текущей версии

- Редактирование ФИО, email и других полей сотрудника пока сделано через API, но не вынесено в отдельную форму админки.
- Удаление сущностей заменено статусами `active`, `inactive`, `suspended`, чтобы не терять историю проходов.
- Поле `schedule` для рабочих часов уже есть в API, но проверка расписаний пока не включена в интерфейс.
- QR payload выводится текстом. Для следующего шага можно добавить отображение PNG или генерацию QR прямо в браузере.
