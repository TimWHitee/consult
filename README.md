# SKUDik Bot 🤖

Телеграм-бот системы контроля и управления доступом (СКУД). Выдаёт QR-пропуска сотрудникам и гостям, верифицирует личность через ФИО и паспортные данные.

---

## Возможности

- **Сотрудник** получает QR-пропуск на текущий день одной кнопкой
- **Гость** получает QR-пропуск по одноразовой ссылке-приглашению от сотрудника
- **QR самодостаточен** — верификация происходит без обращения к БД, только по данным внутри кода
- **Администратор** управляет списком сотрудников через команды в боте
- **Логирование** всех событий в БД
- **Распознавание лица** — заглушка, готовая к подключению face_recognition / DeepFace

## Быстрый старт

### 1. Клонировать и установить зависимости

```bash
git clone <repo>
cd skud_bot
python -m venv venv

# Windows
venv\Scripts\activate
# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Настроить окружение

```bash
cp .env.example .env
```

Открыть `.env` и заполнить:

```env
BOT_TOKEN=ваш_токен_от_BotFather
DB_PATH=skud.db
MASTERPASS_EMPLOYEE=skud15
MASTERPASS_GUEST=skud16
```

### 3. Запустить

```bash
python main.py
```

## База данных

SQLite, файл `skud.db` создаётся автоматически при первом запуске.

| Таблица        | Назначение                                    |
| -------------- | --------------------------------------------- |
| `whitelist`    | Telegram username'ы, допущенные к регистрации |
| `employees`    | Зарегистрированные сотрудники                 |
| `invite_codes` | Одноразовые коды приглашений для гостей       |
| `guests`       | Выданные гостевые пропуска                    |
| `logs`         | Лог всех событий                              |

> ⚠️ При изменении схемы таблиц удалите `skud.db` — он пересоздастся автоматически.

## Формат QR-кода

QR содержит строку вида:

```
Иванов Иван Иванович|4521123456|21.02.2026|a3f9c1d2...
```

Поля: `ФИО | паспорт | дата визита | SHA-256 хеш`

Хеш считается как:

```
sha256(fio_norm + "|" + passport + "|" + date + "|" + masterpass)
```

Где `fio_norm` — ФИО в нижнем регистре без лишних пробелов. Проверяющее устройство считает хеш самостоятельно из полей QR и сравнивает — БД не нужна.

## Роли и сценарии

### Администратор (tg_id = 896079043)

Первый запуск:

1. Написать `/start` боту
2. Нажать «Поделиться контактом» — регистрируется автоматически, минуя whitelist

Управление сотрудниками:

```
/addemployee @username    — добавить сотрудника в whitelist
/removeemployee @username — удалить сотрудника из whitelist
/admin                    — список команд
```

---

### Сотрудник

Первый запуск:

1. Администратор добавляет username через `/addemployee @username`
2. Сотрудник пишет `/start` боту
3. Нажимает «Поделиться контактом» — username проверяется по whitelist

Получить QR на сегодня:

- Нажать **«🪪 Мой QR-пропуск»**

Пригласить гостя:

1. Нажать **«👤 Пригласить гостя»**
2. Ввести ФИО гостя (как в паспорте)
3. Ввести дату визита (ДД.ММ.ГГГГ)
4. Получить одноразовую ссылку — переслать гостю

---

### Гость

1. Перейти по ссылке от сотрудника (`t.me/bot?start=КОД`)
2. Нажать **«🎫 Получить QR-пропуск гостя»**
3. Ввести ФИО (должно совпасть с тем, что указал сотрудник)
4. Ввести серию и номер паспорта (10 цифр)
5. Получить QR-пропуск — показать при входе

> Ссылка одноразовая. Пропуск действителен только в день, указанный сотрудником.

## Распознавание лица

Отправьте боту фотографию — бот сохранит её в папку `face_photos/`.

Подключение реализации в `handlers/face.py`, функция `handle_face_photo`. Варианты:

```python
# Вариант 1: face_recognition (локально)
import face_recognition
result = face_recognition.compare_faces([known_encoding], unknown_encoding)

# Вариант 2: DeepFace
from deepface import DeepFace
result = DeepFace.verify(img1_path, img2_path)

# Вариант 3: внешний API (Face++, Azure Face API и др.)
response = requests.post(API_URL, files={"image": open(filepath, "rb")})
```

## Конфигурация

| Параметр              | Описание                            | По умолчанию |
| --------------------- | ----------------------------------- | ------------ |
| `BOT_TOKEN`           | Токен бота от BotFather             | —            |
| `DB_PATH`             | Путь к файлу SQLite                 | `skud.db`    |
| `MASTERPASS_EMPLOYEE` | Соль для хеша пропусков сотрудников | `skud15`     |
| `MASTERPASS_GUEST`    | Соль для хеша пропусков гостей      | `skud16`     |
| `QR_DIR`              | Папка для сохранения QR-файлов      | `qr_codes`   |
| `ADMIN_TG_ID`         | Telegram ID администратора          | `896079043`  |

> `MASTERPASS_*` — секретная соль, которая хранится только на боте и на проверяющем устройстве. Менять её нужно одновременно на обеих сторонах.
